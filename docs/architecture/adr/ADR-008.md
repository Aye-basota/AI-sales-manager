# ADR-008: AI-Automation Rate Tracking

## Status

Accepted

## Context

The customer and team need a quantitative answer to “How many dialogs are handled by the AI without human intervention?” This metric guides prioritization of automation improvements and identifies when operator handoff is too frequent.

## Decision

Add a boolean column `was_escalated` to the `conversations` table. Any operator-driven status change through `PUT /conversations/{id}/status` sets `was_escalated = true`. Provide `GET /analytics/automation-rate` that returns total conversations, AI-handled count, escalated count, and percentage.

This definition intentionally treats any human operator interaction as escalation, which is conservative and easy to explain to stakeholders.

## Consequences

- Positive: simple, accurate metric derived directly from the existing conversation table.
- Positive: no additional tracking service required; works with current analytics endpoint.
- Negative: future manual takeovers that do not use the status endpoint would not be captured; if such a feature is added, it must also set `was_escalated`.
