# ADR-001: Asynchronous Health Check Design

## Status

Proposed

## Context

The `/health` endpoint must remain responsive even when the database or scheduler is under load.

## Decision

Use an asynchronous health check that inspects scheduler state without blocking on long-running operations.

## Consequences

- Positive: fast response time supports QR-001.
- Negative: health check may report `degraded` rather than detailed failure reasons.
