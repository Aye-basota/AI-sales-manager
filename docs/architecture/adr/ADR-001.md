# ADR-001: LLM Output Guardrails

## Status

Proposed

## Context

The system generates outbound messages on behalf of real Telegram accounts. Unsafe or off-brand content can damage the client's reputation or trigger account bans.

## Decision

Implement a guardrails layer (`app/llm/guardrails.py`) that checks LLM output for forbidden topics, markdown symbols, CJK/Arabic characters, bot self-identification, and repetition before the message is sent.

## Consequences

- Positive: blocks unsafe content before delivery, supporting QR-01.
- Negative: guardrails can reject legitimate messages if thresholds are too strict; tuning is required.
