# ADR-006: Funnel Upload and Preview API

## Status

Accepted

## Context

MVP v1 supported funnel configuration only through the `Script` model and Admin Bot. Customer feedback requested the ability to upload funnel definitions from files or markdown. The team also needs a safe way to preview a funnel before attaching it to a campaign.

## Decision

Introduce a dedicated `Funnel` entity with:

- `app/models/funnel.py` — persistence model.
- `app/schemas/funnel.py` — request/response schemas.
- `app/services/funnel_parser.py` — parsers for `json` and `text` formats with validation (stages list, unique names, required fields).
- `app/api/funnels.py` — `POST /api/funnels/preview` (non-persistent) and `POST /api/funnels/upload` (persistent, with overwrite guard for running campaigns).

The parser normalizes every stage to a common schema so the rest of the system can consume custom funnels the same way as default funnels.

## Consequences

- Positive: operators can version and share funnel definitions outside the application.
- Positive: preview endpoint prevents broken funnels from reaching running campaigns.
- Negative: adds a new table and migration; running-campaign overwrite guard requires explicit `force=true`.
