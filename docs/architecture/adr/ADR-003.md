# ADR-003: Scheduler-Driven Outbound Processing

## Status

Proposed

## Context

The system must process campaigns every 5 minutes, selecting accounts and sending messages within a bounded time to avoid job stacking.

## Decision

Use APScheduler with a periodic `process_campaigns()` task. Account selection, LLM generation, and message dispatch are executed asynchronously and bounded by daily limits and rate limits.

## Consequences

- Positive: bounded processing cycle supports QR-03.
- Negative: heavy LLM latency can push the cycle close to its limit; retries and fallbacks are required.
