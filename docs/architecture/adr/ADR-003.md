# ADR-003: Global Exception Handling

## Status

Proposed

## Context

The API receives requests from frontends and third-party integrations that may contain malformed JSON or missing fields.

## Decision

Use a global exception handler to catch validation errors and return `400 Bad Request` with descriptive messages.

## Consequences

- Positive: prevents crashes and improves debuggability, supporting QR-003.
- Negative: error messages must be reviewed to avoid leaking internal details.
