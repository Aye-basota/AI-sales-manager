# ADR-005: External Prompt Configuration and Versioning

## Status

Accepted

## Context

LLM prompts are a high-churn part of the product. Customer feedback on MVP v1 showed that the first messages felt too salesy, and the team expects frequent A/B testing of prompt wording. Embedding prompts directly in `app/llm/prompts.py` forces a code change, review, and redeploy for every wording tweak and mixes prompt content with orchestration logic.

## Decision

Move all prompt templates, language/emoji rules, nurturing rules, and intent-classification prompt into an external JSON file (`app/config/prompts/v1.json`). Load the configuration at startup via a cached loader (`app.config.prompts.load_prompt_config`) and use safe template formatting so missing placeholders do not crash the application.

## Consequences

- Positive: prompt changes become configuration changes; version is tracked in `version` field; faster iteration and safer rollback.
- Positive: prompt content is reviewable as structured data rather than Python f-strings.
- Negative: startup depends on the config file being present; a missing or invalid file raises an error early (this is acceptable because the failure is obvious).
