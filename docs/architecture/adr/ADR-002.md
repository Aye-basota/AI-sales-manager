# ADR-002: Scheduler Health Integration

## Status

Proposed

## Context

The product relies on APScheduler for campaign processing. Operators need to distinguish between a healthy system and one where scheduling has stopped.

## Decision

Expose scheduler running state through the `/health` endpoint, returning `ok` when running and `degraded` when not.

## Consequences

- Positive: simple availability proxy supports QR-002.
- Negative: does not capture all possible scheduler failure modes.
